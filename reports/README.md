# Reports Folder

This folder contains all cybersecurity reports in JSON format.

## File Structure

- **`reports-index.json`**: Auto-generated index of all reports (do not edit manually)
- **`template.json`**: Template for creating new reports
- **`informe-*.json`**: Individual report files

## Creating a New Report

1. Copy `template.json`
2. Rename it to a descriptive name (e.g., `wannacry-analysis.json`)
3. Fill in all required fields
4. Run the index generation script:
   - **Node.js**: `npm run generate-index` (from project root)
   - **PowerShell**: `.\generate-index.ps1` (from project root)
   - **GitHub Actions**: Just push the file to GitHub

## Report Format

All reports MUST be in English. The web application will translate the UI elements, but the report content remains in English.

```json
{
  "title": "Report Title",
  "source": "Organization Name",
  "description": "Detailed description of the threat, vulnerability, or incident",
  "url": "https://link-to-full-report.com",
  "cve": "CVE-2024-12345",
  "categories": ["Category1", "Category2"],
  "date": "15-01-2024"
}
```

### Field Descriptions

| Field | Required | Type | Description |
|-------|----------|------|-------------|
| `title` | ✅ | string | Report title in English |
| `source` | ✅ | string | Organization that published the report (e.g., "CISA", "Kaspersky") |
| `description` | ✅ | string | Summary or detailed description in English |
| `url` | ❌ | string | Link to the full report (optional) |
| `cve` | ❌ | string | Associated CVE identifier (optional) |
| `categories` | ✅ | array | List of relevant categories/tags |
| `date` | ✅ | string | Publication or update date (**dd-mm-yyyy** format, Spanish) |

### Suggested Categories

Use English category names from this list (or create your own):

- `APT` - Advanced Persistent Threats
- `Malware` - Malicious software
- `Ransomware` - Encryption-based extortion
- `Phishing` - Social engineering attacks
- `Vulnerability` - Security flaws
- `Web Security` - Web application vulnerabilities
- `Critical Infrastructure` - Industrial/critical systems
- `Espionage` - Cyber espionage activities
- `Supply Chain` - Supply chain attacks
- `Data Breach` - Data leaks or breaches
- `Zero-Day` - Unpatched vulnerabilities
- `Healthcare` - Healthcare sector threats
- `Finance` - Financial sector threats
- `IoT` - Internet of Things vulnerabilities
- `Mobile` - Mobile device threats
- `Cloud` - Cloud security issues

## Important Notes

⚠️ **Do NOT edit `reports-index.json` manually**
- This file is auto-generated by scripts
- Manual changes will be overwritten

⚠️ **Do NOT delete `template.json`**
- The generation scripts exclude this file automatically
- It serves as a reference for creating new reports

✅ **Files to commit to Git**
- Your new report JSON files
- Let GitHub Actions handle `reports-index.json`
